# ניטור זיהום מי שתייה — תמונה כוללת

> עדכון אפריל 2026

---

## מטרה

מסמך זה מתאר את מערך הכלים לניטור, ניתוח וניהול סיכונים של זיהום מי שתייה. המערכת מורכבת מארבעה פרויקטים משלימים, המכסים את מחזור חיי המידע — מאיסוף נתוני שטח, דרך זיהוי מגמות, ועד לדירוג סיכון מרחבי ופרשנות גיאו-כימית.

## פרויקטים

### 1. כלי הממרה — `convert_report_to_intake`

**מטרה:** המרת טפסי ניטור שטח של אתרי דלק לפורמט intake של מערכת המידע של הרשות.

**מקור הנתונים:** דוחות שדה מאתרי דלק ("אתרי דלק") — קבצי Excel.

**שני רכיבים:**
- ממשק שורת פקודה: `convert_report_to_intake.py` (~935 שורות)
- אפליקציית Streamlit: `app.py` (~465 שורות)

**קובץ פרמטרים:** `param_table.xlsx` — 830 פרמטרים.

**מבנה קלט:**
- כותרת אתר: שורות 1–11 (שם אתר, תאריך, מעבדה, דוגם)
- מדידות: שורות 13 ואילך, עד 20 בארות לטופס

**פלט:**
- קובץ Excel בפורמט intake (9 עמודות)
- קובץ דוח שגיאות נפרד

**רזולוציית קוד באר:**
שלושה שלבים עוקבים: זיכרון (`well_codes_memory.csv`) ← חיפוש אינטראקטיבי ← שגיאה.  
הזיכרון מתעדכן אוטומטית ומפתח לפי (שם אתר, שם באר).

**התאמת שמות מטושטשת:**
שמות מעבדה ודוגמים מוצלבים מול רשימה רשמית באמצעות `difflib.SequenceMatcher`.

**זיהוי חריגות היסטוריות:**
אזהרה כאשר ערך נוכחי שונה פי 100 או חלקי 100 מהמדידה הקודמת.

**ולידציה:**
- **שגיאות** — חוסמות את השורה, חייבות לתיקון לפני ייצוא
- **אזהרות** — אינפורמטיביות, אינן חוסמות
- **זיהוי אוטומטי** של שני פורמטי נתונים היסטוריים: כותרות בשורה 1 לעומת שורה 5

### 2. WQ Trend Monitor — ניטור מגמות איכות מים

**מטרה:** זיהוי מוקדם של מגמות עלייה בריכוז מזהמים בבארות ניטור, תוך דירוג לפי רמת דחיפות.

#### ארכיטקטורה

```
עיבוד מקדים Python
       ↓
payload_v6_final.json  (675KB, 1037 קומבינציות, 536 בארות)
       ↓
דשבורד HTML/JS עצמאי
```

המערכת מתוכננת לריצת לילה (nightly batch) ואינטגרציה עם פורטל GIS.

#### ציר ניתוח 1 — ריכוז יחסי לתקן

אחוז ריכוז ממוצע (5 שנים אחרונות) מתוך תקן איכות מים.  
ערכי סף: <30% / 30–80% / ≥80%.

#### ציר ניתוח 2 — מגמת Mann-Kendall + Theil-Sen

- **חלון:** 5 שנים (נהלי) + סדרה מלאה (אינפורמטיבית)
- **Theil-Sen Slope:** שיפוע חציוני של כל זוגות (%תקן/שנה)
- **SNR Gating:** SNR = |שיפוע| / std_שארית; חסך SNR<0.3 → MK מוסתר (ממשיך לחשב)
  - ≥1.0 = חזק, 0.3–1.0 = בינוני, <0.3 = חלש
- מגמה מוגדרת כ-UP רק כאשר: MK מובהק (p<0.05) + SNR≥0.3

#### ציר ניתוח 3 — Soft Trigger

3 מדידות אחרונות (5 שנים בלבד) עולות בהחלטיות — ללא תלות ב-MK.

#### ציר ניתוח 4 — Pettitt Change-Point Detection (חדש ב-v6)

חיפוש פרוגרסיבי בזנב הסדרה:
1. 5 שנים אחרונות
2. שליש אחרון
3. חצי שני
4. סדרה מלאה

**קריטריון:** p < 0.1 — שומר את ה-CP האחרון (לא המובהק ביותר).  
**כיוון:** UP / DOWN / FLAT על פי חציון לפני/אחרי (×1.1 / ×0.9).  
**CP_UP** = אות פעיל לסיווג WATCH.

#### רמות סיווג — 4 דרגות

| דרגה | סמל | תנאים |
|------|-----|--------|
| **ALERT** | 🔴 | ≥80% + (מגמה↑ **או** trigger) |
| | | 30–80% + (מגמה↑ **וגם** trigger) |
| | | 30–80% + מתקרב לתקן (std ≤5yr) |
| **WATCH** | 🟡 | ≥80% + ללא אות |
| | | 30–80% + כל אות (MK/trigger/CP_UP) |
| | | <30% + כל אות |
| **OK** | 🟢 | נתונים מספיקים, ללא אותות |
| **Excluded** | ⚪ | n5=0, n<4, אין גילויים |

**התפלגות נוכחית (v6):** 8 ALERT · 38 WATCH · 490 OK

#### מגבלות ידועות

- **צנזור:** אפסים וערכי '<' → ε=0.001; ערכי LOD אמיתיים נאבדים לסטטיסטיקה
- **עונתיות:** אין תיקון עונתי
- **Pettitt:** עמיד לזיהוי בבארות תנודתיות בחלון 5 שנים
- **n5=1,2:** ללא MK ו-trigger; מסתמך על ריכוז + CP היסטורי בלבד

#### מפת דרכים — 4 שלבים

| שלב | תוכן |
|-----|------|
| **Phase 1** | גרפי אגרגציה (עוגה/דונאט לפי דרגה/אגן/פרמטר) + בנייה מחדש עם payload_v6_final |
| **Phase 2** | סטטוס תפעולי של בארות, קבוצות מזהמים (~830 פרמטרים), קואורדינטות בארות |
| **Phase 3** | הוצאת תצורת תקנים לקובץ חיצוני (~100+ תקנים), תצוגת מפה, pipeline לילי אוטומטי |
| **Phase 4** | ניתוח מגמה פוסט-CP, סימון נתונים חשודים, שכבת UI פשוטה |

### 3. דירוג סיכון מרחבי — Spatial Risk Model

**מטרה:** קביעת רמת הסיכון של כל באר הפקה בהתאם לקרבתה למקורות זיהום וזמן ההגעה הצפוי.

#### עקרונות עיצוב

- **Lightweight:** חישובים במצב יציב (Steady-State) לפי חלון זמן — ללא סימולציות הידראוליות כבדות
- **Conservative/Worst-Case:**
  - `b_eff` ≈ 15m — שאיבת שכבה דקה סלקטיבית
  - `n_eff` ≈ 5% — זרימה מהירה בנתיב עדיפות

#### סכמת קלט

**טבלאי:**
| שכבה | תוכן |
|------|------|
| `Wells_Prod` | 36 חודשי שאיבה → Q_max, Q_avg |
| `WQ_History` | היסטוריית איכות מים — בארות הפקה |
| `Config` | קבועי מודל (K, i, n_eff, b_eff, D_max, ועוד) |

**GIS:**
| שכבה | תוכן |
|------|------|
| `Sites` | מקורות זיהום (אתרים מסווגים) |
| `Plumes_Mapped` | נוצות זיהום מיפוי |
| `Heads_Raster` | גובה מי תהום (לחישוב גרדיאנט) |

#### יישום דו-שלבי

**שלב A — Critical Risk Alert (פלט מהיר)**
- חישוב גיאומטריית אזור לכידה ל-2 שנים (מודל wide-drop):
  - `L_s = Q / (2π·K·b_eff·i)`
  - `W_well = π·L_s`
  - `L_up = (K·i / n_eff) · t`
- חיתוך מרחבי: `Capture_2y ∩ (Sites ∪ Plumes_Mapped)` → דגל Critical
- **פלט:** Quick Look Report — רשימת בארות קריטיות לטיפול מיידי

**שלב B — Full Risk Model**
- הרחבת אזורי לכידה ל-t=5 ו-t=10 שנים
- תחזית התקדמות נוצה: `L_eff = min(v·t, ריקבון כימי, D_max)`
  - רוחב מינימלי: 50m
  - עומק: עד עובי האקוויפר
- ניתוח מגמת MK על `WQ_History` של בארות הפקה — מגמה עולה מוסיפה לציון הסיכון
- **סיווג סיכון:**

| רמה | תנאי |
|-----|------|
| Critical | חפיפה עם אזור לכידה 2 שנים |
| High | חפיפה עם אזור לכידה 5 שנים |
| Medium | חפיפה עם אזור לכידה 10 שנים |
| Low | ללא חפיפה |

#### פלטים סופיים

- **שכבת `Production_Risk`:** שכבת נקודות עם שדה `Risk_Level`, סימבולוגיה (אדום/כתום/צהוב/ירוק)
- **שכבות גיאומטריה:** `Capture_Zones` + `Predicted_Plumes` (הקשר ויזואלי)
- **אפליקציית ווב:** דשבורד + סימולטור What-If (מחוונים n_eff, b_eff → עדכון זמן-אמת)
- **דוח שנתי:** ניתוח מלא + השוואת YoY

#### הבחנה: MK ב-WQ Trend Monitor לעומת Spatial Risk Model

| כלי | בארות | MK על |
|-----|-------|-------|
| WQ Trend Monitor | ניטור | ריכוז מזהמים — אות חיצוני |
| Spatial Risk Model | הפקה | WQ_History — אות פנימי |

שני השימושים משלימים ואינם כפולים.

### 4. GeoForensics — פרשנות גיאו-כימית

**מטרה:** ניתוח PCA ואשכולות לזיהוי מקורות זיהום ותחומי השפעה.

כלי ניתוח שמספק פרשנות של ממצאי ה-WQ Trend Monitor — מאפשר הבחנה בין רקע גיאוכימי טבעי לבין זיהום אנתרופוגני.

> **הערה:** הלוגיקה של PCA נותרת לא-מתועדת מחוץ לקוד עצמו.

## ארכיטקטורה כוללת

```
                 מערכת המידע של הרשות (DB)
                      ↑              ↓
              כלי הממרה       WQ_History / Wells_Prod
           (טפסי שדה אתרי דלק)        ↓
                              WQ Trend Monitor  ←→  דירוג סיכון מרחבי
                              (בארות ניטור)          (בארות הפקה)
                                       ↑
                              GeoForensics (פרשנות אופציונלית)
```

**זרימת מידע:**
1. `כלי הממרה` קולט טפסי שדה מאתרי דלק → ממיר לפורמט intake → מעלה ל-DB
2. `WQ Trend Monitor` קורא `WQ_History` של בארות ניטור → מזהה מגמות → פלט JSON/HTML
3. `דירוג סיכון מרחבי` קורא `WQ_History` ו-`Wells_Prod` של בארות הפקה + שכבות GIS → דירוג מרחבי
4. `GeoForensics` מנתח מגמות מ-WQ Trend Monitor → פרשנות גיאו-כימית אופציונלית
